一、项目简介
黑马点评是一个前后端分离的本地生活服务平台，对标大众点评，聚焦本地商户、探店笔记、用户互动、优惠券秒杀四大核心板块，核心目标是通过实战掌握Redis在企业级项目中的各类应用场景，解决高并发下的缓存、锁、限流等常见问题。
项目核心价值：脱离单纯的技术demo，模拟真实业务场景，将Redis多种数据结构、缓存设计模式、分布式锁等技术与业务深度结合，解决缓存穿透、击穿、雪崩、秒杀超卖、一人一单等企业高频面试问题，提升后端开发实战能力。
二、技术栈详情
2.1 后端技术栈
- 框架：SpringBoot 2.7.x（稳定版，适配多数企业开发环境）
- 持久层：MyBatis-Plus（简化CRUD操作，提升开发效率）
- 数据库：MySQL 8.0（支持新特性，适配高并发场景）
- 缓存中间件：Redis 6.x（核心，覆盖多种数据结构实战）
- 身份认证：JWT（无状态登录，适配分布式场景）
- 构建工具：Maven 3.6+（依赖管理、项目构建）
- 服务器：Tomcat（内置，无需额外部署）
- 工具类：Apache Commons Lang3、FastJSON（数据处理、JSON转换）
2.2 前端技术栈
- 框架：Vue 2.x（主流前端框架，易上手、生态完善）
- UI组件库：ElementUI（快速搭建前端页面，适配后台管理、用户端）

- 其他：HTML5、CSS3、JavaScript（基础前端技术）
2.3 核心技术亮点（面试重点）
- Redis全场景实战：熟练运用String、Hash、Set、ZSet、List、BitMap、GEO等数据结构，适配不同业务场景
- 缓存设计与优化：实现Cache Aside缓存旁路模式，解决缓存一致性问题
- 缓存三大问题解决方案：针对性解决缓存穿透、缓存击穿、缓存雪崩，落地企业级优化方案
- 分布式锁：基于Redis+Lua脚本、Redisson实现分布式锁，解决并发安全问题
- 高并发秒杀：实现优惠券秒杀功能，解决超卖、一人一单、接口限流等核心问题

三、项目核心功能
3.1 用户模块
- 短信验证码登录：对接短信接口（课程模拟实现，无需真实对接第三方），实现手机号验证码发送、验证，完成登录，同时支持登录状态保持

- 用户信息管理：查询、修改个人信息，基于Redis缓存用户信息，提升查询效率
- 用户注销：课程新增功能，实现JWT令牌失效、缓存清理，完成用户注销操作
3.2 商户模块
- 商户信息查询：查询商户详情、列表，支持多条件筛选，基于Redis缓存商户数据，解决高并发查询问题
- 商户分类：商户按行业分类，缓存分类数据，减少数据库查询压力
- 附近商户：基于Redis GEO数据结构，根据用户地理位置查询附近商户，按距离排序
- 商户推荐：课程新增功能，基于用户浏览历史、商户热度，实现商户智能推荐，结合Redis ZSet存储热度排名
3.3 用户互动模块
- 用户签到：基于Redis BitMap实现每日签到，统计连续签到天数，发放签到奖励
- 好友关注：用户关注/取消关注博主，查询粉丝列表、关注列表，实现共同关注功能（基于Redis Set交集）
- 探店笔记：发布、浏览探店笔记，笔记点赞、取消点赞（基于Redis Set/ZSet），实现点赞排行榜
- 笔记评论：课程新增功能，支持用户对探店笔记发表评论、回复评论，基于Redis List存储评论列表，实现评论分页查询
3.4 优惠券与秒杀模块
- 优惠券管理：发布普通优惠券、秒杀优惠券，设置使用规则、有效期，补充课程中讲解的优惠券分类（满减券、折扣券）、领取限制、使用范围限制功能
- 高并发秒杀：实现优惠券秒杀功能，通过Redis+Lua脚本保证原子操作，防止超卖，完全对照课程中秒杀流程、Lua脚本编写、并发测试的内容实现
- 一人一单：基于分布式锁实现“一人一单”限制，避免重复下单，课程重点讲解Redisson分布式锁的使用，以及锁的超时、重入机制
- 接口限流：针对秒杀接口，基于Redis计数器实现限流，防止接口被压垮，补充课程中讲解的令牌桶限流方案，适配不同接口的限流需求
- 优惠券领取与核销：课程核心功能，实现优惠券领取（基于Redis Set防止重复领取）、核销流程，关联订单与优惠券，实现核销后优惠券失效逻辑

五、项目启动步骤
请严格按照以下步骤操作，避免因配置错误导致项目无法启动：
5.1 数据库初始化
1. 启动本地MySQL服务，打开MySQL客户端（Navicat、SQLyog等），连接数据库
2. 创建数据库，执行以下SQL语句（编码格式为utf8mb4）：
        CREATE DATABASE hmdp DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
3. 导入项目内数据库脚本：找到项目 src/main/resources/db 目录下的 hmdp.sql 文件，在创建的 hmdp 数据库中执行该脚本，完成表结构和测试数据的初始化
5.2 配置文件修改
找到项目 src/main/resources 目录下的 application.yml 配置文件，修改以下核心配置（根据自身环境调整）：
spring:
  # 数据库配置
  datasource:
    url: jdbc:mysql://localhost:3306/hmdp?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root  # 你的MySQL用户名（默认root）
    password: 123456  # 你的MySQL密码（替换为自己的密码）
    driver-class-name: com.mysql.cj.jdbc.Driver
  # Redis配置
  redis:
    host: localhost  # Redis服务器地址（本地默认localhost）
    port: 6379  # Redis端口（默认6379）
    password:  # Redis密码（无密码则留空，有密码则填写对应密码）
    database: 0  # Redis数据库索引（默认0）
    timeout: 10000ms  # 连接超时时间
# JWT配置（可根据需求调整）
jwt:
  secret: hmdp123456  # JWT密钥（自定义，建议复杂一些）
  expiration: 86400000  # JWT有效期（单位：ms，默认24小时）
# 服务器配置
server:
  port: 8080  # 后端端口（默认8080，可修改，需与前端配置一致）
5.3 依赖下载与项目启动
1. 用IDEA打开项目，IDEA会自动识别Maven项目，等待Maven自动下载依赖（若下载缓慢，可切换阿里云Maven镜像源）
2. 启动本地Redis服务：确保Redis服务正常运行（可通过Redis客户端连接测试）

六、项目核心难点与解决方案
项目核心难点集中在高并发场景下的缓存优化、并发安全问题，以下是具体难点及落地解决方案：
核心难点
解决方案
核心思路
缓存穿透
布隆过滤器 + 缓存空值
用布隆过滤器过滤不存在的key，避免请求穿透到数据库；对不存在的key缓存空值，设置短期过期时间
缓存击穿
互斥锁（Redis+Lua） + 逻辑过期
热点key过期时，用互斥锁保证只有一个请求去数据库查询，其他请求等待；或给热点key设置逻辑过期，避免key真正过期
缓存雪崩
过期时间随机化 + 多级缓存 + 熔断降级
给缓存key设置随机过期时间，避免批量过期；引入本地缓存+Redis缓存的多级缓存；服务熔断降级，避免缓存失效导致数据库压垮
秒杀超卖
Redis+Lua脚本原子操作
用Lua脚本实现“查询库存 + 扣减库存”的原子操作，避免并发下的超卖问题
一人一单
分布式锁（Redisson）
以“用户ID+优惠券ID”为key创建分布式锁，保证同一用户只能下单一次，避免重复下单
接口高并发限流
Redis计数器限流
基于Redis Incr命令统计单位时间内的请求次数，超过阈值则拒绝请求，保护接口

八、常见问题与排查方案
项目启动或运行过程中，可能遇到以下问题，可按对应方案排查：
8.1 Redis相关问题
- 问题1：Redis连接失败，控制台报“Could not connect to Redis”
- 排查：① 检查Redis服务是否启动；② 检查application.yml中Redis的host、port、password是否正确；③ 关闭本地防火墙，避免端口被拦截；④ 确认Redis允许本地连接（修改redis.conf配置）。
- 问题2：Redis操作报错，报“NOAUTH Authentication required”
- 排查：Redis设置了密码，但application.yml中未配置Redis密码，补充配置redis.password即可。
8.2 数据库相关问题
- 问题1：数据库连接失败，报“Access denied for user 'root'@'localhost'”
- 排查：检查application.yml中MySQL的username、password是否正确，确认MySQL服务正常运行。
- 问题2：执行SQL脚本报错，报“Unknown column 'xxx' in 'field list'”
- 排查：MySQL版本过低，部分语法不支持，建议升级到MySQL8.0；或重新导入sql脚本，确保脚本完整。
8.3 项目启动相关问题
- 问题1：Maven依赖下载失败，报“Could not find artifact xxx”
- 排查：切换阿里云Maven镜像源，在pom.xml中添加阿里云镜像配置，或手动下载依赖导入本地仓库。
- 问题2：项目启动报错，报“Port 8080 was already in use”
- 排查：8080端口被其他程序占用，可修改application.yml中server.port为其他端口（如8081），或关闭占用8080端口的程序。
- 问题3：前端页面无法访问后端接口，报“跨域问题”
- 排查：项目已配置全局跨域拦截器，若仍出现跨域，检查前端请求地址是否正确（需与后端端口一致），或重启后端项目。
8.4 业务功能相关问题
- 问题1：秒杀时出现超卖
- 排查：确认Redis+Lua脚本是否正确，分布式锁是否生效，检查秒杀业务逻辑中的库存扣减逻辑，确认是否按照课程讲解的“先缓存扣减、后数据库同步”流程实现。
- 问题2：用户签到数据统计错误
- 排查：确认Redis BitMap的操作是否正确（对照课程中BitMap的set、getbit、bitcount用法），连续签到天数的计算逻辑是否有误，检查签到奖励发放的触发条件。
- 问题3：笔记评论无法分页或显示异常
- 排查：确认Redis List存储评论的逻辑是否正确，分页查询时的range命令使用是否规范，对照课程中评论分页的实现代码调整。
- 问题4：优惠券领取失败或重复领取
- 排查：检查Redis Set存储已领取用户ID的逻辑，确认领取限制（每人限领一张）是否生效，对照课程中优惠券领取的核心代码排查。
